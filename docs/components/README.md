## Format (pr. komponent)
Opret en fil: `docs/components/<komponent>.md`

Indhold bør som minimum være:
- Hvad er det?
- Hvornår bruger vi den?
- Props/API (hvis relevant)
- Eksempler
- Kendte begrænsninger
- Link til relaterede beslutninger (ADR) hvis relevant
